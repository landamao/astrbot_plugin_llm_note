# 📓 笔记本插件 (LLM Note)

为 AstrBot 提供持久化的笔记能力，让大模型可以记住用户信息、偏好、关系等，并支持群组全局笔记。

---

## ✨ 功能特性

- **个人笔记**：为每个用户在群组/私聊中独立存储笔记列表（最多 20 条，自动淘汰旧条目）。
- **全局笔记**：群组级别的共享笔记，所有成员都会在上下文中看到。
- **LLM 自动注入**：每次对话前，自动将用户个人笔记和群组全局笔记拼接到 LLM 的 prompt 中，让模型"记住"信息。
- **四款 LLM 工具**（大模型可主动调用）：
  - `edit_note` – 增、删、改笔记内容（批量操作，无索引位移问题）
  - `search_note` – 在群组内搜索包含关键词的笔记
  - `get_note` – 获取指定用户的笔记（需提供用户 ID）
  - `get_user_id` – 通过昵称反查用户 ID
- **用户指令**：
  - `/清空笔记` – 清空当前群/私聊笔记（自动备份，管理员可清群数据，非管理员10分钟冷却）
  - `/恢复笔记 <文件名>` – 从备份文件恢复笔记（非管理员10分钟冷却）
  - `/清空笔记日志` – 清空错误日志文件（仅管理员）
- **隐私保护**：笔记内容仅对 LLM 可见，LLM 可选择性透露给用户。
- **安全防护**：文件操作内置目录穿越拦截，禁止删除核心数据文件。
- **持久化存储**：使用 JSON 文件保存，原子写入防止数据损坏。

## 📦 安装

1. 将本插件克隆或下载到 AstrBot 的 `plugins` 目录下：
   ```bash
   git clone https://github.com/landamao/astrbot_plugin_llm_note.git
   ```
2. 重启 AstrBot，插件将自动加载。

## 🔧 配置

无需额外配置，开箱即用。

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `自动删除时长` | int | 24 | 清空笔记时备份文件多少小时后自动删除，0代表永不删除 |

## 🧠 使用方式

### 1. 自动注入

当用户发送消息时，如果该用户存在个人笔记或群组存在全局笔记，插件会在发送给 LLM 的请求中自动追加如下格式的提示：

```
<note>
【动态记忆与私密笔记协议】
本文是你的私密笔记，仅对你可见，你可以选择性透露给用户

这是你为当前用户记录的笔记本内容（格式：[索引] 内容\n\n）：
[0] 用户喜欢喝美式咖啡
[1] 生日是 5 月 20 日

这是当前全局区的笔记内容：
[0] 本项目截止日期为 2026-07-01
</note>
```

LLM 将会知晓这些信息，并在回答时参考。

---

### 2. LLM 工具详解

大模型可以调用以下工具（通常由对话触发，用户无需手动输入）。

#### 🛠️ `edit_note`

**用途**：对当前用户（或全局）的笔记列表进行批量编辑。

**参数**：
- `user_id` (string, 可选) – 要操作的用户 ID，不填则默认当前用户；若传入 `"global"` 则操作群组全局笔记。
- `operations` (array) – 操作列表，每个操作包含：
  - `action` (string) – `"add"`、`"delete"` 或 `"replace"`
  - `index` (number) – 目标索引（0-based），可为负数（表示倒数）
  - `content` (string, 可选) – 对 `add`/`replace` 必填

**返回值**：操作后的完整笔记列表（或清空提示），如有部分操作出错会附带错误信息。

> **批量操作无索引位移**：在一次调用中同时进行删除、插入等操作时，所有操作基于原始索引执行，无需担心索引位移问题。内部实现先标记删除/替换（None 占位），再按原始索引顺序重组列表，最后统一清理占位符。

**示例操作**（JSON 格式）：
```json
[
    {"action": "add", "index": 0, "content": "插入到第一条"},
    {"action": "replace", "index": -2, "content": "覆盖**倒数**第二条的内容"},
    {"action": "delete", "index": 0},
    {"action": "delete", "index": 1}
]
```

> **注意**：笔记总数限制为 **20 条**，超出时自动丢弃最旧的条目。

---

#### 🔍 `search_note`

**用途**：在当前群组内搜索所有笔记（含全局笔记），模糊匹配关键词。

**参数**：
- `keyword` (string) – 搜索关键词

**返回值**：匹配到的笔记列表，附带用户昵称（如果有缓存）和用户 ID。

---

#### 📄 `get_note`

**用途**：获取指定用户的笔记内容（需提供用户 ID）。

**参数**：
- `user_id` (string) – 目标用户的 ID

**返回值**：该用户的笔记列表。

---

#### 👤 `get_user_id`

**用途**：通过用户昵称（部分匹配）查找对应的用户 ID。

**参数**：
- `user_name` (string) – 要查找的用户名

**返回值**：匹配到的用户 ID 及昵称列表（可能多个）。

---

### 3. 用户指令

#### 🗑️ `/清空笔记`

清空当前群组或私聊的笔记，执行前自动备份到数据目录。

**权限规则**：
- **私聊**：任何用户可清空自己的笔记
- **群聊**：仅管理员可清空群数据（含全员笔记 + 全局笔记）

**用法**：
```
/清空笔记          # 清空当前群/私聊的笔记
/清空笔记 <群ID>   # 管理员清空指定群的笔记
```

**自动清理**：备份文件默认 24 小时后自动删除（可通过配置调整，0 代表永久保留）。

执行后会生成备份文件（如 `data_群1103659691删除前数据_2026-06-20_06-02-30.json`），并提示恢复方式。

---

#### ♻️ `/恢复笔记 <文件名>`

从备份文件恢复笔记数据。

**权限规则**：
- 管理员可恢复任意群数据（含跨群恢复）
- 非管理员仅可恢复自己的私聊数据
- 非管理员恢复操作有 10 分钟冷却

**用法**：
```
/恢复笔记 data_群1103659691删除前数据_2026-06-20_06-02-30.json
```

## 💬 典型对话示例

> **用户**：帮我记一下我喜欢喝冰美式，少冰。  
> **机器人**：（调用 `edit_note` 添加笔记）已记录你的喜好：你喜欢喝冰美式，少冰。

> **用户**：我上次让你记的我的生日是哪天？  
> **机器人**：（自动注入笔记后回答）你的生日是 5 月 20 日。

> **用户**：群里的"小王"是谁？  
> **机器人**：（调用 `get_user_id` 查找）用户小王的 ID 是 123456789。

> **用户**：添加全局笔记：项目 deadline 是 7 月 1 日。  
> **机器人**：（以 `global` 身份调用 `edit_note`）已更新全局笔记。

> **用户**：/清空笔记  
> **机器人**：清空成功，若要恢复，请使用`/恢复笔记 data_群1103659691删除前数据_2026-06-20_06-02-30.json`

## ⚠️ 注意事项

- **私聊限制**：全局笔记（`user_id="global"`）仅在群组中有效，私聊中无法使用。
- **用户缓存**：用户昵称仅在对话过程中缓存，机器人重启后需重新获取（通常通过新消息自动填充）。
- **数据文件**：`data.json` 位于插件数据目录，请勿手动修改以免破坏格式。
- **错误日志**：`error.log` 记录插件运行时的错误信息，可用 `/清空笔记日志` 清空。
- **备份文件**：清空笔记时自动生成的 `data_*.json` 备份文件也存放在数据目录，默认 24 小时后自动删除，可安全手动删除。
- **安全性**：笔记内容未加密，但仅存储在本地，确保服务器安全即可。文件操作内置目录穿越拦截。
